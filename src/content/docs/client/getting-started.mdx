The Relay client is a modern desktop chat application built with Electron. It connects to self-hosted Relay servers, giving you complete control over your chat experience while maintaining a sleek, user-friendly interface.

## Download and Installation

### System Requirements

<List>

    <ListItem>**Linux**: Ubuntu 18.04+, Debian 10+, or equivalent</ListItem>
</List>

## Initial Setup

### First Launch

When you first launch Relay, you'll see the home screen with no servers connected. The interface includes:

-   **Sidebar**: Navigation between servers and settings
-   **Main area**: Server list and connection status
-   **Breadcrumb**: Current location indicator

### Joining Your First Server

To connect to a Relay server, you'll need:

<List>
    <ListItem>
        **Server URL**: The address of the Relay server (e.g.,
        `http://192.168.1.100:8080`)
    </ListItem>
    <ListItem>
        **Username**: Your desired username (must be unique on the server)
    </ListItem>
    <ListItem>**Nickname**: Your display name (can be changed later)</ListItem>
    <ListItem>
        **Invite Code**: A valid invite code from the server administrator
    </ListItem>
</List>

#### Step-by-Step Connection Process

<List type="ordered">
    <ListItem>
        **Click "Join Server"** button on the home screen or in the sidebar
    </ListItem>
    <ListItem>
        **Fill in the connection details:** - **Server URL**: Enter the complete
        URL including protocol (http:// or https://) - **Username**: Choose a
        unique identifier (alphanumeric characters recommended) - **Nickname**:
        Your display name that others will see - **Invite Code**: Paste the
        invite code provided by the server admin
    </ListItem>
    <ListItem>**Click "Join Server"** to initiate the connection</ListItem>
    <ListItem>
        **Wait for connection**: The client will generate cryptographic keys and
        authenticate with the server
    </ListItem>
</List>

<Callout type="success">
    Once connected successfully, you'll be automatically redirected to the
    server's main channel and can start chatting immediately!
</Callout>

### Understanding Server Connection

#### Connection Status Indicators

-   **Green dot**: Server is online and connected
-   **Red dot**: Server is offline or unreachable
-   **"Connecting..."**: Attempting to establish connection
-   **"Retrying"**: Automatically retrying failed connections

## Managing Multiple Servers

### Server List Organization

Relay supports connecting to multiple servers simultaneously:

-   **Drag and drop** servers in the sidebar to reorder them
-   **Server icons** display custom server branding when available
-   **Connection status** is shown for each server individually

### Server Management Actions

Right-click on any server in the sidebar to access:

<List>
    <ListItem>
        **Export Identity**: Save your cryptographic identity for backup
    </ListItem>
    <ListItem>**Retry Connection**: Manually retry failed connections</ListItem>
    <ListItem>**Leave Server**: Disconnect and remove the server</ListItem>
</List>

## Troubleshooting Common Issues

### Connection Problems

<Tabs defaultTab="url-issues">
  <TabList>
    <TabTrigger id="url-issues">URL Issues</TabTrigger>
    <TabTrigger id="network-issues">Network Issues</TabTrigger>
    <TabTrigger id="invite-issues">Invite Code Issues</TabTrigger>
  </TabList>

  <TabContent id="url-issues">
    **Incorrect Server URL Format:**
    - Ensure you include the protocol: `http://` or `https://`
    - Include the port if not using standard ports: `http://server.com:8080`
    - Verify the server address with your administrator

    **Common URL Examples:**
    ```
    http://localhost:8080          # Local server
    http://192.168.1.100:8080     # Local network server
    https://chat.example.com      # Public server with SSL
    ```

  </TabContent>

  <TabContent id="network-issues">
    **Firewall/Network Blocking:**
    - Check if your firewall allows the Relay client
    - Verify network connectivity to the server
    - Try connecting from a different network to isolate issues

    **Corporate Networks:**
    - WebSocket connections may be blocked
    - Contact your IT administrator about allowing Relay traffic
    - Consider using HTTPS servers which are more likely to work

  </TabContent>

  <TabContent id="invite-issues">
    **Invalid Invite Code:**
    - Verify the code was copied correctly (case-sensitive)
    - Check if the invite has expired
    - Confirm the invite hasn't reached its usage limit
    - Request a new invite from the server administrator
  </TabContent>
</Tabs>
